# Lab 00 — Kickoff

Welcome to **AI Analytics Skills for BI Analysts**.

## What you will learn

By the end, you can open a BI workspace, give an AI coding assistant the right files and constraints, ask for an evidence-backed analysis, and verify the result before it reaches a dashboard, catalog page, or stakeholder update.

This workshop is not about prompting a chatbot. It is about packaging repeatable analytics work as **context-aware, tool-using, reviewable AI workflows**.

## The core loop

Every lab repeats the same loop:

```
Open context
  → ask with constraints
  → inspect evidence
  → propose a change or artifact
  → validate
  → communicate caveats
```

AI is valuable here because it helps you structure the work, inspect evidence, reduce blind spots, and create artifacts a human can review — not because it writes more words.

## The lab arc

1. **Find it** — Lab 02: Trace Tableau lineage with workspace context.
2. **Tune it** — Lab 03: Triage SQL Server performance safely.
3. **Govern it** — Lab 04: Repair catalog context with AI (the full agentic stack).
4. **Prove it** *(optional)* — Lab 05: Prove the KPI still reconciles.

Lab 01 sets up your workspace. Lab 06 is the verification checklist you apply to every artifact.

## Each lab follows the same rhythm

- **Watch** — the facilitator shows the move.
- **Try** — you run the workflow in your own workspace.
- **Verify** — you compare the output against evidence and the checklist.
- **Document** — you produce a small, reviewable artifact.

## How to run today

Pick an entry mode for the room:

1. **Guided mode** — step by step (default for a mixed-experience room).
2. **Context dump** — fast track for experienced participants.
3. **Best guess** — facilitator makes assumptions and the room corrects.
